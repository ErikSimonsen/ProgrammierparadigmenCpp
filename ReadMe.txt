Im Ordner Playground können Sie herumspielen und sich etwas mit SVN vertraut machen.
Löschen Sie anschließend möglichst wieder alle Dateien, die Sie in den Ordner
Playground eingecheckt haben.
Herr Helmke will diese Dateien nicht auf seiner Platte haben.

In Aufgabe* legen Sie bitte im Ordner den Code und Sonstiges für die Aufgabe ab.
Dieses umfasst die Projektdateien, sofern VisualStudio verwendet wird, u.a.:
Hier sind es in der Regel 
   - die Source-Code-Dateien *.cpp oder *.cxx
   - die Header-Dateien *.h
   - Die Visual-Studio-Dateien *.sln, *.vcxproj.filters, *.vcxproj.filters
   - Datendateien, die Sie zum Testen erstellt haben

*.exe, *.obj bitte nicht mit einchecken, dafür zieht Herr Helmke ansonsten Punkte ab.
In den Ordnern befindet sich eine batch-Datei delAuto.bat für Windows, die ab dem Ordner,
von wo aus sie aufgerufen wird (und dann auch in den Unterordnern), viele unnötige 
Dateien löscht.
Der versteckte Ordner .vs  und der Order x64 oder entsprechend für 32 Bit mit den 
   Unterordnern Debug bzw. Release dürfen ebenfalls nicht mit eingecheckt werden

Der Ordner Images enthält Screen-Dumps, die zeigen sollen, dass Sie Ihr Programm 
 ausgeführt haben.

Im Ordner Changes sollen Sie Änderungen protokollieren, die Sie nach dem finallen 
  Checkin noch durchgeführt haben, damit die Ausgaben auf unbekannten Eingabedateien
  so wie gefordert, aussehen, z.B. können Pfade angepasst werden oder 
  schwerwiegender wird ein "kleiner" in ein "kleiner gleich" umgewandelt.
  Sind die Änderungen nicht protokolliert, wird Ihr Programm so bewertet,
  als ob gar keine Änderungen durchgeführt wurden. In wie weit die
  Änderungen zu Punktabzug führen, wird hinterher entschieden.

Legen Sie in den Ordnern Aufgabe* auch jeweils eine Datei Fertig.txt ab.
Sie darf leer. Sie zeigt an, dass Sie fertig sind und dieser Code für die 
Bewertung verwendet werden soll.
Die Datei muss vor dem Abgabeende eingecheckt werden.

Führen Sie anschließend die Bewertung Ihrer Aufgaben entsprechend
 den nach dem Abgabestichtag veröffentlichten Bewertungskriterien durch.
Tragen Sie Ihre Bewertung in die Datei Bewertung.txt ein und checken diese
24 Stunden nach dem Abgabedatum ebenfalls jeweils im Ordner Aufgabe* ein.
Der Inhalt der Datei Bewertung.txt enthält Ihre Bewertung im jeweils für
jede Aufgabe veröffentlichten Format.
Die Dateien Fertig.txt und Bewertung.txt gibt es noch nicht. Sie werden
von Ihnen erstellt.
